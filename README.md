## 📝 开发规范

1. 场景文件(.tscn)存放在 scenes 目录
2. 脚本文件(.gd)存放在 scripts 目录
3. 资源文件(图片、音频等)存放在 resources 目录
4. UI相关场景和资源存放在 ui 目录
5. 卡牌配置文件存放在 resources/cards 目录

## 🤖 AI助手使用指南

### 基本原则
1. AI助手会严格遵循配置指南中的语法和规则
2. 所有修改都会经过安全性和合理性验证
3. 涉及系统架构的修改需要明确授权

### 禁止事项
1. 不接受违反基础规则的修改请求
2. 不执行可能破坏系统架构的操作
3. 不创建超出现有框架的新机制
4. 不修改核心系统文件结构

### 有效的请求方式
```
# 良好示例
修改：basic_deck.txt
目的：平衡性调整
操作：将攻击牌伤害从1点调整为1.2点
原因：基础伤害值过低

# 不良示例
修改所有卡牌伤害
添加新的卡牌类型
修改核心系统文件
```

### AI助手权限
1. **可以执行的操作**：
   - 卡牌数值调整
   - 卡牌效果修改
   - 新卡牌添加
   - 配置文件格式化
   - 语法错误修正

2. **需要确认的操作**：
   - 批量修改
   - 删除操作
   - 系统配置变更
   - 新机制添加

3. **禁止的操作**：
   - 核心系统修改
   - 框架结构变更
   - 违反规则的修改
   - 未经验证的实验性功能

### 异常处理流程
1. 发现问题时，AI会：
   - 说明具体问题
   - 提供合规的替代方案
   - 请求进一步指导

2. 对于不确定的请求：
   - 要求澄清具体需求
   - 提供可行的方案建议
   - 说明潜在的风险

### 最佳实践
1. **请求格式**：
   ```
   操作：[具体操作类型]
   目标：[目标文件或组件]
   修改：[详细的修改内容]
   原因：[修改原因]
   ```

2. **修改确认**：
   - AI会展示修改预览
   - 说明修改影响
   - 等待确认后执行

3. **反馈循环**：
   - AI执行操作后会提供反馈
   - 用户确认结果
   - 必要时进行调整
